HTML Form Elements and Attributes

  • <form> element: used to create an HTML form for user input.
  • action attribute: used to specify the URL where the form data should be sent.
  • method attribute: used to specify the HTTP method to use when sending the form data. The most common methods are GET and POST.
  • <input> element: used to create an input field for user input.
  • type attribute: used to specify the type of input field. Ex. text, email, number, radio, checkbox, etc.
  • placeholder attribute: used to show a hint to the user to show them what to enter in the input field.
  • value attribute: used to specify the value of the input. If the input has a button type, the value attribute can be used to set the button text.
  • name attribute: used to assign a name to an input field, which serves as the key when form data is submitted. For radio buttons, giving them the same name groups them together, so only one option in the group can be selected at a time.
  • size attribute: used to define the number of characters that should be visible as the user types into the input.
  • min attribute: can be used with input types such as number to specify the minimum value allowed in the input field.
  • max attribute: can be used with input types such as number to specify the maximum value allowed in the input field.
  • minlength attribute: used to specify the minimum number of characters required in an input field.
  • maxlength attribute: used to specify the maximum number of characters allowed in an input field.
  • required attribute: used to specify that an input field must be filled out before submitting the form.
  • disabled attribute: used to specify that an input field should be disabled.
  • readonly attribute: used to specify that an input field is read-only.
  • <label> element: used to create a label for an input field.
  • for attribute: used to specify which input field the label is for.
  • Implicit form association: inputs can be associated with labels by wrapping the input field inside the <label> element.
  • Explicit form association: inputs can be associated with labels by using the for attribute on the <label> element.
  • <button> element: used to create a clickable button. A button can also have a type attribute, which is used to control the behavior of the button when it is activated. Ex. submit, reset, button.
  • <fieldset> element: used to group related inputs together.
  • <legend> element: used to add a caption to describe the group of inputs.
  • Focused state: this is the state of an input field when it is selected by the user.

Working with HTML Table Elements and Attributes

  • Table element: used to create an HTML table.
  • Table Head (<thead>) element: used to group the header content in an HTML table.
  • Table Row (<tr>) element: used to create a row in an HTML table.
  • Table Header (<th>) element: used to create a header cell in an HTML table.
  • Table body (<tbody>) element: used to group the body content in an HTML table.
  • Table Data Cell (<td>) element: used to create a data cell in an HTML table.
  • Table Foot (<tfoot>) element: used to group the footer content in an HTML table.
  • <caption> element: used to add a title of an HTML table.
  • colspan attribute: used to specify the number of columns a table cell should span.

Working with HTML Tools

  • HTML validator: a tool that checks the syntax of HTML code to ensure it is valid.
  • DOM inspector: a tool that allows you to inspect and modify the HTML structure of a web page.
  • Devtools: a set of web developer tools built directly into the browser that helps you debug, profile, and analyze web pages.

Accessibility & Advanced HTML Summary

  1. Multimedia Accessibility
    • <track> element: A self-closing tag used inside <video> or <audio> to provide timed text tracks.
      • src: Path to the .vtt file.
      • kind: Defines the type of text (e.g., subtitles, captions, descriptions, chapters).
      • srclang: Language of the track (e.g., en, fi).
      • label: The title shown to the user in the selection menu.
  2. WAI-ARIA (Accessible Rich Internet Applications)
    • role attribute: Defines the semantic purpose of an element when the tag itself doesn't provide it (e.g., role="button" on a div).
    • aria-label: Provides a string of text to label an element that has no visible text.
    • aria-hidden="true": Hides decorative or redundant elements from assistive technologies.
    • aria-describedby: References the ID of another element that provides a detailed description of the current element.
  3. Semantic Structure
    • <figure> & <figcaption>: Used to group an image or media with its descriptive caption.
    • <aside>: Represents content that is indirectly related to the main content (like sidebars or pull-quotes).
    • Landmark Roles: Implicit roles like <main>, <nav>, <section>, and <footer> that help users navigate page regions.
  4. Form Accessibility Best Practices
    • Label Association: Linking <label for="ID"> with <input id="ID">.
    • Form Validation: Using the required attribute to communicate mandatory fields.
    • Visual Cues: Using <span>*</span> with aria-hidden="true" so the asterisk is visible but ignored by screen readers.
  5. The POUR Principles (WCAG)
    • Perceivable: Users must be able to see or hear the content.
    • Operable: Users must be able to navigate the interface (keyboard, mouse, etc.).
    • Understandable: Users must understand how to use the interface.
    • Robust: The code must work across different browsers and assistive tools.

Specificity values are calculated as a four-part value (a, b, c, d):

  • a: Inline styles (1 or 0).
  • b: Number of ID selectors.
  • c: Number of class selectors, attribute selectors, and pseudo-classes.
  • d: Number of type selectors, pseudo-elements, and universal selectors.

The !important Rule:

It is a CSS keyword used to override all other styling rules for a specific property. It has the absolute highest priority and completely ignores selector specificity (IDs, classes, etc.).

Example:

p { color: blue !important; } — This text will be blue even if there is a more specific ID selector trying to make it red.

Key Rule: Use it only as a last resort (for example, when you can't change an external CSS library). Overusing it makes debugging your code a nightmare because it breaks the natural cascade.

The Cascade Algorithm

The Cascade is the engine that decides which CSS rules win when multiple styles target the same element. It follows this priority logic:

  1. Relevance: Does the selector match the element? Are media queries (device size) active?
  2. Origin & Importance: Direct styles from the author (you) beat browser defaults. The !important flag beats everything.
  3. Specificity: The more specific the selector (IDs > Classes > Tags), the more "weight" it has.
  4. Order of Appearance: If everything else is equal, the rule written last in the code wins.

The inherit Keyword

The inherit value forces an element to take the property value from its parent element, even if that property isn't inherited by default.

  • Best Use: Forcing form elements (button, input) to use the same font or color as the rest of the page.
  • Danger: If used globally (e.g., on the * selector) for properties like margin or border, it can create a "nesting doll" effect and break your layout.

The LVHA Rule: CSS Pseudo-classes Order

To ensure link styles work correctly and don't override each other, follow the LVHA mnemonic (often remembered as "Love Fears Hate Always" or "Lord Voldemort Has Arrived"):

  1. :link — Matches unvisited links.
  2. :visited — Matches links the user has already clicked.
  3. :hover — Triggered when the mouse pointer is over the link.
  4. :active — Triggered at the exact moment the link is being clicked.
Note: :focus is usually placed right before or after :hover to ensure keyboard accessibility.

Mastering CSS Backgrounds

To control how images behave behind the content, we use a set of specific properties. Here’s the breakdown:

  • background-image — The source of the image.
    • Syntax: url("image-path.jpg"). It’s the "what" of your background.
  • background-repeat — Determines if the image tiles (repeats).
    • Options: repeat (default), no-repeat (one single image), repeat-x (horizontal), repeat-y (vertical).
  • background-size — Controls the "scale" of the image.
    • cover: Fills the entire container (might crop edges).
    • contain: Shows the whole image (might leave empty space).
    • auto: Original size.
  • background-position — Moves the image within the container.
    • Values: center, top right, bottom left, or exact pixels/percentages (e.g., 50% 50%).
  • background-attachment — Determines if the image scrolls with the page.
    • scroll: Image moves with the content (default).
    • fixed: Image stays stuck in the background while content slides over it (creates a cool Parallax effect).
  • background — The shorthand property to combine multiple background values into one line.
    • Syntax: [color] [image] [repeat] [attachment] [position] / [size].
    • Crucial Rule: The size (e.g., cover) must come after position and be separated by a forward slash /.
    • Benefit: Saves space and keeps your CSS clean. If a value is missing, it resets to its default.

CSS Gradients

Gradients let you display smooth transitions between two or more specified colors. They are treated as images by CSS.

  • linear-gradient — Creates a transition along a straight line.
    • Syntax: linear-gradient(direction, color1, color2, ...)
    • Direction: Can be an angle (e.g., 45deg) or a keywords (e.g., to right, to bottom left).
    • Color-stops: You can specify where each color starts/ends using percentages (e.g., red 10%, blue 90%).
  • radial-gradient — Creates a transition that radiates from a central point (an elliptical or circular shape).
    • Syntax: radial-gradient(shape size at position, color1, color2, ...)
    • Shape: circle or ellipse (default).
    • Position: Where the "center" of the gradient is (e.g., at center, at top left, or exact pixels).

Important Details:

  1. Multiple Colors: You aren't limited to two colors. You can list as many as you want: linear-gradient(to right, red, orange, yellow, green).
  2. Performance: Since gradients are generated by the browser, they don't require downloading an image file, making your site load much faster.
  3. Background Property: Gradients are set using the background or background-image property, not background-color.

CSS Borders, Outlines, and Rounded Corners

These properties define the visual boundaries of an element and its shape.

  • border — Sets a visible frame around an element. It is part of the box model and takes up space.
    • Syntax: border: [width] [style] [color];
    • Common Styles:
      • solid: A single continuous line.
      • dashed: A series of short line segments (dashes).
      • dotted: A series of small dots.
      • double: Two parallel solid lines.
  • outline — Draws a line outside the border edge. Unlike borders, outlines do not take up space and don't affect the layout.
    • Syntax: outline: [width] [style] [color];
  • border-radius — Defines the radius of the element's corners.
    • Values: Pixels (20px) or percentages (50%).

Design Terms

Layout
Layout is how the visual elements are arranged on a page or screen to communicate a message. These elements may include text, images, and white space. The layout is like the blueprint of a design. Designers must consider the placement, size, and hierarchy of each element.
Alignment
Alignment is how the elements are placed in relation to one another. Using alignment correctly is helpful for making the design look clean and organized. Designers create visual order by aligning elements along imaginary lines, edges, or a central point.
Composition
Composition is the art of arranging elements to create a harmonious design. It determines how elements like images, text, and shapes relate to each other and contribute to the design in an artistic way. While layout mostly focuses on the placement of the elements, composition also considers the artistic impact that this placement will have in the overall design.
Hierarchy
Hierarchy establishes the order of importance of the elements in a design. It's about making sure the most important information is noticed first. You can implement a visual hierarchy with size, color, contrast, alignment, white space, and even typography.
Contrast
Contrast is helpful for guiding user attention to what you want to emphasize. You can do this through variations in color, size, shape, texture, or any other visual characteristic. Strong contrast is also helpful for improving readability.
White Space
White Space, also known as "negative space", is the empty space in a design. It's the area surrounding the elements. You might be surprised to know that white space is not necessarily white. Actually, it can be space in any color or texture. Its purpose is to improve the readability and enhance the visual hierarchy of a design.

These are general design concepts that you will find very often, but you may also find design terms that are more closely related to software development.

The user interface, also known as UI, is how humans interact with computers. A user interface includes the visual and interactive elements that users can see on their screens, like icons, images, text, menus, links, and buttons.

The user experience, also known as UX, is about how users feel when using a product or service. An application with a well-designed user experience is intuitive, easy to use, efficient, accessible, and enjoyable. The user interface plays a key role in making the user experience as easy and enjoyable as possible, so they are very closely related.

Deferred and Lazy Registration

Definition: Design patterns that delay the account creation process until it is absolutely necessary for the user to proceed.

  • Deferred Registration: Allows users to access the core features of a site (e.g., filling a shopping cart or applying filters) and only prompts for registration at the final step, such as during checkout.
  • Lazy Registration: The system automatically creates a "temporary" account for the user. All actions are saved, and the user can convert it into a permanent account later by simply adding a password and email.

Why use it?

  • Lower Friction: Reducing barriers to entry increases the likelihood that a user will stay and engage with the product.
  • Psychology of Ownership: Once a user has invested time into customizing their experience, they are more likely to register to avoid losing their progress.
  • Trust Building: You provide value to the user before demanding their personal data.

Design Briefs and Developer Workflow

Definition: A Design Brief is a project management document that outlines the core objectives, scope, and expectations of a design project. It serves as a roadmap for both designers and developers.

What’s inside a Brief?

  • Target Audience: Who the product is for.
  • Project Goals: What problems the website should solve.
  • Technical Constraints: Specific platforms, browsers, or performance requirements.
  • Brand Guidelines: Colors, fonts, and tone of voice.

How Developers work with them:

  1. Alignment: Developers use the brief to ensure the code structure supports the long-term goals of the project.

  4. Testing: The brief serves as a checklist during the final audit to ensure the site meets all initial requirements.

Absolute Units in CSS

Definition: Fixed-size length units that do not change based on the screen size or user settings. They provide precise control but lack flexibility in responsive design.

  • Pixels (px): The most common unit in web design. Theoretically defined as 1/96th of an inch, but physically depends on the screen's pixel density (PPI).
  • Physical Units (cm, mm, in): Measurements like centimeters or inches. They are rarely used in web development and are mainly intended for print layouts.
  • Points and Picas (pt, pc): Traditional typography units. 1pt is 1/72 of an inch. Mostly used when styling pages for printers.

Why use them?

  • Precision: Ideal for small details like borders (1px) or shadows where you need exact thickness.
  • Fixed Layouts: Sometimes used for "hard" constraints, like a maximum width of a container to prevent it from stretching too far on giant monitors.

Relative Units in CSS

Definition: Units that scale based on another value, such as the parent element's size, the root font size, or the browser viewport. They are the foundation of responsive design.

  • Percentages (%): Define size as a proportion of the parent container.
    • Use case: Creating fluid grids (width: 50%) and responsive images (max-width: 100%).
  • em: Relative to the font size of the element itself (or its parent).
    • Use case: Modular components like buttons. If you change the font size, the padding and margins in 'em' will scale proportionally.
  • rem (Root em): Relative only to the font size of the root element (html).
    • Use case: Best practice for typography. It respects user browser settings, making the site accessible for people with visual impairments.
  • vh & vw (Viewport Units): Relative to the browser window (viewport) size. 1vh = 1% of height, 1vw = 1% of width.
    • Use case: Creating full-screen sections (height: 100vh) or typography that scales smoothly with the window width.

Key Takeaways:

  • Accessibility: Use rem for font sizes instead of pixels so users can scale text in their browser settings.
  • Hierarchy: While 'em' depends on the closest parent (can be tricky with nesting), 'rem' is always predictable because it only looks at the root.
  • Fluidity: Percentages are great for widths, but height in % can be tricky if the parent doesn't have a defined height.
  • Mobile Constraints: Be careful with vh on mobile devices, as the address bar appearing/disappearing can cause sudden layout shifts.

CSS calc() Function

Definition: A built-in function that allows you to perform calculations (+, -, *, /) directly in CSS to determine property values dynamically.

  • Dynamic Sizing: Combine different units like calc(100% - 50px). Perfect for layouts where one part is fluid and another is fixed.
  • Mathematical Operators: Supports addition (+), subtraction (-), multiplication (*), and division (/).
  • Syntax Rule: Operators (especially + and -) must be surrounded by whitespace to be valid.

Best Practices:

  • Zero Units: Unlike regular CSS, inside calc() even a 0 must have a unit (e.g., 0px).
  • Multiplication/Division: At least one operand must be a unitless number (e.g., calc(10px * 2)).

CSS Pseudo-classes

Definition: Special keywords added to selectors that allow styling elements based on their specific state or structural position within a parent element.

  • User Interaction:
    • :hover — Triggered when the user mouses over an element.
    • :active — Applied while an element (like a button) is being clicked.
    • :focus — Applied when an element is selected (e.g., via keyboard or clicking an input).
  • Structural Selection:
    • :first-child — Selects an element if it is the very first child of its parent.
    • :last-child — Selects an element if it is the very last child of its parent.
  • Link & Form States:
    • :visited — Styles links that have already been clicked by the user.
    • :disabled — Styles interactive elements (inputs, buttons) that are set to a disabled state.

Best Practices:

  • Visual Feedback: Use pseudo-classes like :hover to provide immediate feedback to the user, making the UI feel "alive".
  • Accessibility: Always ensure :focus states are clearly visible for users who navigate via keyboard.

Action & State Pseudo-classes

Definition: Advanced pseudo-classes used to style elements based on focus flow, form availability, and URL navigation.

  • :focus-within: Applies styles to a parent element if any of its children receive focus. Great for highlighting entire forms.
  • :enabled / :disabled: Specifically targets form elements based on whether they are interactive or locked.
  • :target: Styles an element that matches the current URL fragment (e.g., the specific section linked via an ID like #about).

Key Takeaway:

  • User Guidance: Use :focus-within to improve form usability and :target to create smooth visual cues when navigating via anchor links.

Input Pseudo-classes

Definition: Specific pseudo-classes used to enhance user experience by styling form elements based on interaction, validation, and requirements.

  • Interaction:
    • :focus — Highlights the active field where the user is currently typing.
    • :hover — Provides visual feedback when the cursor is over an input.
    • :checked — Styles selected radio buttons or checkboxes.
  • Validation:
    • :required / :optional — Targets fields based on whether they must be filled to submit the form.
    • :valid / :invalid — Styles inputs based on whether they meet the validation rules (e.g., correct email format).
    • :in-range / :out-of-range — Used for number inputs to check if the value is within the specified min/max limits.
  • System & State:
    • :disabled — Styles elements that cannot be interacted with, often using cursor: not-allowed.
    • :autofill — Targets fields that were automatically filled by the browser's saved data.

Best Practices:

  • Visual Cues: Use :invalid sparingly—it's often better to show the error after the user finishes typing rather than immediately.
  • Clarity: Always use cursor: not-allowed for disabled elements to clearly signal that they are non-interactive.

Location Pseudo-classes

Definition: Selectors that target links and elements based on their relationship with the current document or the user's browsing history.

  • Link States:
    • :link — Selects all anchor elements (<a>) that have not been visited yet.
    • :visited — Selects links the user has already clicked. For privacy reasons, browsers limit the styles you can change here (mostly just color).
    • :any-link — A convenient shorthand that targets any element that would match either :link or :visited.
  • Navigation & Fragments:
    • :target — Styles a specific element when its ID matches the hash in the URL (e.g., highlighting a section when jumping to it via #section1).
    • :local-link — Designed to style links pointing to the same document (internal links), though currently lacking browser support.

Key Takeaway:

  • Navigation Clarity: Using :target is a great "no-JS" way to highlight the active section of a long page, helping users stay oriented.
  • Visual History: Always style :visited slightly differently to help users remember which pages they've already read.

Tree-structural Pseudo-classes

Definition: Selectors that target elements based on their precise location or relationship within the HTML document tree (DOM).

  • Root & Content:
    • :root — Matches the highest-level parent (usually <html>). Primarily used for defining global CSS variables.
    • :empty — Targets elements that have no children at all (not even text nodes), useful for hiding broken or empty list items.
  • Position-based (Counting):
    • :nth-child(n) / :nth-last-child(n) — Selects elements by their index from the start or end. Supports numbers, odd, even, or formulas like 2n+1.
    • :only-child — Selects an element only if it is the unique child of its parent.
  • Type-specific:
    • :first-of-type / :last-of-type — Selects the first or last instance of a specific tag among its siblings.
    • :nth-of-type(n) — Similar to nth-child, but only counts elements of the same type (e.g., only <p> tags).
    • :only-of-type — Selects an element if it's the only one of its kind inside the parent, even if other different tags are present.

Best Practices:

  • Precision: Use :nth-of-type instead of :nth-child if your container has mixed tags (like h1, h2, and p) to avoid selecting the wrong element.
  • Clean UI: Use :empty { display: none; } to automatically clean up your layout from empty generated elements.

Functional Pseudo-classes

Definition: Advanced selectors that accept arguments (selectors) inside parentheses to perform complex matching and logic.

  • Grouping & Logic:
    • :is(s1, s2) — Reduces repetition by grouping multiple selectors into one. The specificity is determined by its most specific argument.
    • :where(s1, s2) — Identical to :is(), but always has zero specificity, making it easy to override later.
    • :not(selector) — Selects elements that do not match the argument. Perfect for excluding specific items from a general style.
  • The "Parent" Selector:
    • :has(selector) — Targets an element if it contains at least one descendant matching the argument. It allows styling parents based on their children's existence or state.

Best Practices:

  • Code Cleanliness: Use :is() to keep your stylesheets DRY (Don't Repeat Yourself).
  • Layout Control: Use :has() to change a card's layout only when it contains an image or a specific icon, avoiding unnecessary class toggling in JS.

CSS Pseudo-elements

Definition: Virtual elements used to style specific parts of an element or insert "synthetic" content without modifying the actual HTML structure.

  • Content Insertion:
    • ::before — Inserts content at the very beginning of the element's content.
    • ::after — Inserts content at the very end of the element's content.
    • Note: Both require the content property to function.
  • Text & UI Styling:
    • ::first-letter — Targets the first character of a block-level element (e.g., for drop caps).
    • ::marker — Styles the bullet points of list items (<li>) or summary elements.
    • ::selection — Styles the portion of the document that has been highlighted/selected by the user.
    • ::placeholder — Targets the hint text inside <input> and <textarea> elements.
  • Advanced Hooks:
    • ::first-line — Styles only the first line of text in an element.
    • ::spelling-error — Allows customization of the visual indicator for misspelled words.

Best Practices:

  • Syntax: Always use the double colon (::) to distinguish pseudo-elements from pseudo-classes (like :hover).
  • Decorative Use: Keep ::before and ::after for decorative purposes (icons, quotes). Crucial information should remain in the HTML for accessibility.
  • Brand Identity: Customizing ::selection and ::placeholder to match your brand colors makes the UI feel much more polished.
